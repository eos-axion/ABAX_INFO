Module 1 — Performance

Core Web Vitals & Vitesse Mobile

Les Core Web Vitals sont les metriques officielles de Google pour mesurer l'experience utilisateur. Depuis 2021, ils influencent directement le classement dans les resultats de recherche. Notre scan utilise l'API Google PageSpeed Insights (Lighthouse) en mode mobile.

LCP — Largest Contentful Paint
~3.8s
Qu'est-ce que c'est ? Le temps pour afficher le plus grand element visible (souvent l'image hero ou le titre principal).

Bon < 2.5s Moyen < 4s Lent > 4s

Chez ABAX : Le rendu JavaScript cote client (Wix Thunderbolt) retarde l'affichage du contenu principal. Le navigateur doit d'abord charger puis executer le JS avant d'afficher quoi que ce soit.
FCP — First Contentful Paint
~3.0s
Qu'est-ce que c'est ? Le temps avant que le premier element (texte ou image) apparaisse a l'ecran. C'est la premiere impression de vitesse.

Bon < 1.8s Moyen < 3s Lent > 3s

Chez ABAX : Wix charge un framework JavaScript lourd (Thunderbolt) avant d'afficher le premier pixel. Le FCP est penalise par cette architecture CSR (Client-Side Rendering).
CLS — Cumulative Layout Shift
~0.18
Qu'est-ce que c'est ? Mesure la stabilite visuelle. Un CLS eleve = les elements bougent pendant le chargement (agacant pour l'utilisateur).

Bon < 0.1 Moyen < 0.25 Mauvais > 0.25

Chez ABAX : Le chargement dynamique du contenu via JavaScript cause des decalages visuels. Les images et blocs se repositionnent apres le rendu initial.
INP — Interaction to Next Paint
~320ms
Qu'est-ce que c'est ? Le temps entre une interaction utilisateur (clic, touche) et la reponse visuelle. Remplace le FID depuis mars 2024.

Bon < 200ms Moyen < 500ms Lent > 500ms

Chez ABAX : Le framework Wix et les scripts tiers (analytics, chat) bloquent le thread principal, retardant la reponse aux clics.

💡 Pourquoi c'est important : Google utilise ces metriques pour classer les sites. Un LCP > 2.5s entraine une penalite directe dans les resultats mobiles. 53% des visiteurs quittent un site qui met plus de 3 secondes a charger. Wix impose un plancher de performance que vous ne pouvez pas optimiser sans changer de plateforme.

Le SEO On-Page, c'est ce que Google lit en premier pour comprendre votre site. Chaque balise a un role precis. Notre scan verifie les elements les plus impactants du code source de votre page d'accueil. Attention : sur Wix, le HTML est genere dynamiquement en JavaScript — les IA generatives ne peuvent souvent pas le lire.

📝 Blog & Contenu editorial
Inactif

Probleme : 8 mois sans publication. Google considere votre site comme inactif. Vos concurrents publient 2 a 4 articles par mois.
📌 Donnees structurees Schema.org
Absent
A quoi ca sert ? Schema.org permet aux moteurs de comprendre votre entreprise (nom, adresse, services) dans un format standardise. Il active les "rich snippets" dans Google (etoiles, FAQ, etc.).
Probleme : Aucun schema detecte. Pas de LocalBusiness, pas de ITService, pas de FAQ. Les moteurs IA ne comprennent pas ce que fait ABAX Info.
🌐 Rendu JavaScript (CSR)
Problematique
A quoi ca sert ? Le mode de rendu determine comment les moteurs accedent a votre contenu. Le SSR (Server-Side Rendering) envoie du HTML lisible immediatement. Le CSR (Client-Side) necessite d'executer du JavaScript.
Probleme : Wix Thunderbolt utilise le CSR. Google doit faire 2 passes pour indexer. ChatGPT, Perplexity et Claude ne peuvent pas du tout lire votre contenu.
🗣️ Hreflang (ciblage geographique)
Absent
A quoi ca sert ? La balise hreflang indique a Google quelle version de la page afficher selon le pays/la langue du visiteur. Essentiel pour les activites multi-marches.
Probleme : ABAX Info est actif en Suisse et en France avec des pages tarifs CHF/EUR, mais aucune balise hreflang. Les pages se cannibalisent dans les resultats.
🏷️ Pages services
Insuffisant
A quoi ca sert ? Chaque service merite sa propre page pour cibler des mots-cles specifiques. "Infogerance PME Geneve" et "Cybersecurite PME" ne doivent pas etre sur la meme page.
Probleme : 1 seule page /services pour tous les metiers IT. Il manque au minimum 8 pages dediees (voir slide constats).

Module 4 — Email Health

Delivrabilite & Securite Email

Ce module verifie la sante de votre domaine email via les enregistrements DNS publics. Un domaine mal configure = emails en spam, usurpation d'identite possible, et perte de confiance de vos clients et partenaires.

7/10
Score Email Health
Correct — A renforcer

Calcul du score :
SPF present (+3) ✅ | SPF strict ~all (+1) ✅
DMARC present (+3) ✅ | DMARC actif p=quarantine (+1) ❌
MX present (+1) ✅ | DKIM (+1) ❌ non verifie
Total : 7/10 — Bon point de depart

SPF — Sender Policy Framework
Qu'est-ce que c'est ? Le SPF dit au monde "seuls ces serveurs ont le droit d'envoyer des emails au nom de abaxinfo.com". C'est la premiere ligne de defense contre le spam et l'usurpation.

Chez ABAX : SPF present et bien configure en mode ~all (softfail). Inclut Outlook 365, Siportal, Autotask et Mailgun. ✅ Bonne pratique.
DMARC — Domain Message Authentication
Qu'est-ce que c'est ? DMARC dit aux serveurs de reception "si un email de @abaxinfo.com echoue les verifications SPF/DKIM, voici quoi faire" (rejeter, mettre en quarantaine, signaler).

Chez ABAX : DMARC present mais en mode p=none (surveillance uniquement). Devrait passer a p=quarantine puis p=reject pour bloquer activement les usurpations.
MX — Mail Exchange + Mailinblack
Qu'est-ce que c'est ? Les enregistrements MX indiquent quels serveurs recoivent les emails pour votre domaine. Mailinblack est un filtre anti-spam professionnel.

Chez ABAX : MX bien configure avec Mailinblack en priorite (filtre anti-spam) puis Outlook 365. ✅ Configuration professionnelle, coherente pour un prestataire IT.

Actions prioritaires

🔴 Urgences (impact immediat)

1

Ajouter Schema.org (LocalBusiness + ITService)

2

Sur toutes les pages, surtout les tarifs. Stoppe la cannibalization et ameliore le ciblage geographique. ⏱ 15 min.

3

Creer 8 pages services dediees

1 page par service IT cle. Chaque page cible un mot-cle specifique (ex: "infogerance PME Geneve"). Potentiel : +250-500 visites/mois.

4

Relancer le blog (1 article/semaine)

Sur vos expertises IT. +30 a 50% de trafic organique en 90 jours. Nourrit aussi les moteurs IA.

5

Renforcer DMARC (p=quarantine)

Passer de p=none a p=quarantine pour bloquer activement les emails usurpes. Protection anti-phishing.

🟠 Optimisations (moyen terme)

6

Ajouter une section FAQ structuree

FAQ schema sur chaque page service. Alimente les "People also ask" de Google et les reponses IA.

7

Creer des temoignages clients structures

Reviews schema avec nom, entreprise, note. Renforce E-E-A-T et les signaux de confiance pour les IA.

8

Evaluer une migration hors Wix

Vers WordPress ou Next.js avec SSR. Elimine les limitations CSR et debloque le plein potentiel SEO/GEO.

9

Optimiser les Core Web Vitals

Images, lazy loading, prefetch. Objectif LCP < 2.5s et CLS < 0.1. Limite par Wix mais ameliorable.

10

Strategie de contenu GEO-first

Publier du contenu pedagogique structure pour etre cite par ChatGPT et Perplexity comme source experte.
